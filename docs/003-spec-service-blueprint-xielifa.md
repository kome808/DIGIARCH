# 文件 3｜數位典藏系統服務藍圖（謝里法捐贈）（更新版 v3.3｜含徵集清單/逐件審議/進度查詢/保存修護）

> 本文件定位：
> 
> 1. 捐贈/徵集審議段服務藍圖（收件→建立清單流水號→評估→契約→入藏審議逐件通過/否決）
> 2. 後續作業段服務藍圖（全宗整理→入庫→保存修護→數位化→編目/審核→權利/索引→上架→前台查詢→提借(閱)）

> 一致性規則：Backstage 模組命名以文件 1（v3.3）Canonial L1 為準：整合查詢／徵集審議作業／全宗架構管理／入庫作業管理／數位化作業／詮釋資料維護／網站管理／提借(閱)作業／資料統計及報表／系統設定。
> 

---

## 1. 泳道（Lanes）

- **L1 使用者行為**：捐贈人/家屬；後續轉為館內人員、民眾/研究者
- **L2 前台接觸點**：網站、表單、Email、電話、窗口
- **L3 Frontstage（館員可見互動）**：回覆、說明、接待、審查回覆
- **L4 Backstage（系統內流程）**：案件、清單流水號、逐件審議、入庫、修護、數位化、編目、權利、索引
- **L5 Support（支援資源）**：清單模板、會議排程、掃描/轉檔設備、修護資源、物流

## 3. 斷點與風險（供規格與 UX 討論）

- 徵集審議階段名稱未知/無數位化：需「流水號清單（名稱可空）」與可控的對外進度顯示
- 整批評估 vs 逐件取捨：需「案件層結論＋逐件通過/否決＋理由」與可匯出
- 契約僅作權利盤點參考：需在權利設定中明確「契約 ≠ 最終權利狀態」
- Schema 欄位集多樣：需「資料類型設定（Schema/Template）」可選用與調整必填/非必填
- Metadata 審核與追溯：需「審核流程＋版本差異 Log」
- 保存分級需連動限制：等級 3/4 自動限制實體外借；並保留等級歷程
- 數位化量級與轉檔：需「批次轉檔/縮圖/解析度檢核/狀態標記」與儲存容量規劃
